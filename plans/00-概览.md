# NovaBot 设计概览

> NOVA 社团智能助手 · 面向语雀知识库的 AI 助手
> 创建时间：2026-03-26
> 状态：设计中

---

## 1. 愿景

**一句话定义**：NovaBot 是 NOVA 社团的共享 AI 助手，以语雀知识库为核心，辅助社团成员高效学习、协作与活动组织。

**类比**：
- Peacebot 是谷和平的个人助理
- NovaBot 是整个社团的"共享助理"

---

## 2. 核心理念

### 2.1 NOVA 社团定位

> 来源：语雀文档《初步设想》《这是介绍》

**社团定位**：思维训练营，不只是"南大的社团"

**核心价值**：
- **元认知能力**：学会拆解自己的学习过程
- **自主学习**：用 AI 赋能学习进程
- **活人感**：真实、有温度、有人味
- **归属感**：不是官方账号，是成员的个人视角

**目标受众**：想要提升元认知能力、学习技能的大学生们

**关键词**：CAC、自主学习、AI 赋能、个人成长

### 2.2 知识共享（NovaWiki 思路）

> 来源：黄亮《项目五选 x》中的 NovaWiki 方案

```
视角转变：作者本位 → 知识本位
    ↓
自动去重：相似内容聚类，合并为"知识卡片"
    ↓
保留个人思考：心路历程作为旁注
    ↓
增量更新：语雀文档变更时，只更新相关部分
```

**痛点**：
- 上学期 CAC 要求学爬虫，大家各自搜索资料，**重复劳动**
- 新学习者难以消化前人留下的大量文章

**设计原则**：
- NovaWiki 和语雀仓库**互补**，不替代
- 每个学习者可以在 NovaWiki 上留下独特的足迹

### 2.3 激发元认知

NOVA 社团强调自学与元认知能力激发。

NovaBot 不应只是"给答案"，而是：

| 传统方式 | NovaBot 方式 |
|----------|-------------|
| 直接给答案 | 给答案 + 指出来源，让用户能追溯 |
| 推荐内容 | 解释为什么推荐，帮用户理解自己的学习路径 |
| 替代思考 | 引导思考，提出延伸问题 |
| 按作者索引 | 按知识主题聚合，去重后呈现核心内容 |

## 3. 社团痛点分析

> 来源：语雀文档调研（2026-03-26）

### 3.1 协作效率低

**现象**：
- 黄亮："上学期我在实践组待了一段时间...然后我直到退社都没找到合适的协作者"
- "不要让 1+N 小于 1+AI"
- 项目"分享出来"不等于"有人来协作"

**根因**：
- 不知道谁在做什么
- 找不到技能/兴趣匹配的人
- 缺乏协作触发机制

**NovaBot 解法**：伙伴推荐、学习小组匹配、技能互补识别

---

### 3.2 知识重复劳动

**现象**：
- "上学期 CAC 要求学爬虫，大家几乎都在自己搜索资料，造成大量重复劳动"
- "新的学习者难以消化前人留下的大量文章"

**根因**：
- 语雀仓库按**作者**组织，不是按**知识主题**
- 同一知识点散落在不同人的文档里
- 缺乏知识地图/索引

**NovaBot 解法**：NovaWiki（知识去重 + 主题聚合）、知识地图可视化

---

### 3.3 运营层面混乱

**现象**（来自《初步设想》）：
- 账号定位不清晰（AI成长？南大社团？认知提升？）
- 内容比例不确定（整活 vs 知识 vs 教程）
- 更新频率难坚持
- 团队分工"谁有空谁做" → 最后没人做
- 账号人格不统一

**根因**：
- 缺乏数据驱动的运营决策
- 没有自动化工具辅助
- 人工运营成本高

**NovaBot 解法**：自动周报、热门内容发现、活跃度统计、运营数据看板

---

### 3.4 知识库管理成本高

**现象**（来自你的经历）：
- 语雀更新推送噪音大（点了发布就通知）
- 不知道改了什么，值不值得看
- 文档检索困难（作者本位 vs 知识本位）

**根因**：
- 推送缺乏智能过滤
- 缺乏变更摘要
- 检索维度单一

**NovaBot 解法**：智能推送（LLM判断变更价值）、变更摘要、RAG检索

---

### 3.5 学习反馈缺失

**现象**：
- 学完了，没有输出 → 白学
- 不知道自己学到什么程度
- 没有人追问"你学会了什么"
- Vibe Coding 容易"技术栈虚浮"（苏语杨）

**根因**：
- 学习过程缺乏记录
- 缺乏输出驱动机制
- 没有进度可视化

**NovaBot 解法**：输出驱动提醒、学习追踪、认知评估、进度可视化

---

### 3.6 归属感与活人感缺失

**现象**（来自《初步设想》《这是介绍》）：
- 社团账号像"官方账号"，没有温度
- 新成员不知道怎么融入
- "这是一个南大的社团"无法成为唯一标签

**根因**：
- 缺乏个人视角的内容
- 成员之间缺乏连接点
- 没有"精神社员"概念

**NovaBot 解法**：成员故事发现、互动促进、匿名提问箱、社群活跃度分析

---

### 3.7 形式主义风险（深层痛点）

> 来源：于粤铭《一些乱七八糟的碎碎念与一些暴论》

**现象**：
- "摆子"能完美满足制度要求：
  - 每两周当一次评议员，到场就行
  - 问题让 AI 生成就好
  - 文档复制粘贴
- "参与者"可能被清退：
  - 勤恳思考项目、记录探索
  - 因时间冲突错过输出活动
  - Table 被约满，分享又没成熟内容
  - 即将面临清退

**根因**：
- 制度用「是否有线下输出」判断「是否有效参与」
- 无法区分形式合规 vs 真实投入
- 探索初期、磨合期没有被包容

**李赫的核心论点**：
> 评判一个社员是否积极参与，不该只看是否有线下输出，而是是否在学在做在分享，以及质量如何。

**形式主义风险**：
- 文档从认知沉淀 → 合规证明材料
- 分享从真实交流 → 周期性打卡
- 评议从互相补全 → 履行制度义务

**NovaBot 解法**：
- **参与质量评估**：不只看是否有输出，看内容质量、推进程度
- **参与形式扩展**：追踪多种参与形式（文档、反馈、协作、推进记录）
- **认知沉淀检测**：判断内容是真实思考还是合规痕迹
- **缓冲机制提醒**：探索期/磨合期时主动提醒，而非直接清退

---

### 3.8 痛点 → 功能映射表

| 痛点 | NovaBot 功能 | 优先级 |
|:-----|:-------------|:-------|
| 协作效率低 | 伙伴推荐、学习小组匹配、技能互补识别 | P1 |
| 知识重复劳动 | NovaWiki（知识去重）、知识地图 | P1 |
| 运营混乱 | 自动周报、热门发现、运营看板 | P2 |
| 知识库管理成本高 | 智能推送、变更摘要、RAG检索 | P1 |
| 学习反馈缺失 | 输出驱动、学习追踪、认知评估 | P1 |
| 归属感缺失 | 成员故事、互动促进、匿名提问箱 | P2 |
| **形式主义风险** | **参与质量评估、认知沉淀检测** | **P1** |

## 5. 功能分层

```
┌─────────────────────────────────────────────────────────────┐
│                      NovaBot                                 │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  🌐 公共层（无门槛，无需绑定）                                 │
│  ├── 知识检索：搜索语雀文档                                  │
│  ├── 智能问答：基于文档回答问题                              │
│  ├── 知识卡片：去重后的主题聚合                              │
│  └── 资源发现：找文档/项目/人                                │
│                                                             │
│  🔐 个人层（绑定语雀账号后解锁）                              │
│  ├── 用户画像：分析你的文档，了解你是什么样的人              │
│  ├── 认知评估：判断你的水平层次                              │
│  ├── 个性化引导：推荐适合你当前阶段的学习路径                │
│  ├── 伙伴推荐：找到兴趣/水平相似的人                         │
│  ├── 订阅推送：你关注的主题/作者更新时通知                   │
│  └── 学习追踪：记录你的成长轨迹                              │
│                                                             │
│  📊 社团层面（定时任务 / 管理触发）                           │
│  ├── 周报：本周热门文档、活跃作者                            │
│  ├── 趋势分析：哪些话题在升温                                │
│  ├── 优质推荐：挖掘被忽视的好内容                            │
│  └── 知识地图：社团知识图谱可视化                            │
│                                                             │
│  💫 人性化层（贯穿各层）                                      │
│  ├── 学习陪伴：来源追溯 + 延伸问题 + 学习建议                │
│  ├── 成长可视化：学习时间线 + 里程碑                         │
│  ├── 情绪感知：识别用户状态，调整语气                        │
│  └── 游戏化：经验值 + 成就系统                               │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```
