# T001: 实现 /bind 指令

> 任务：实现语雀账号绑定功能
> 优先级：P0
> 创建时间：2026-03-26

## 目标

让用户通过 `/bind` 指令绑定语雀账号，支持：
1. Token 验证
2. 姓名/用户名模糊匹配
3. 冲突检测
4. 用户画像生成

## 官方文档要点

来自 `AstrBot开发指导手册-Plugin-Skills-MCP.md`：

1. **指令注册**：使用 `@filter.command("bind")` 装饰器
2. **消息获取**：`event.message_str`、`event.get_sender_id()`
3. **消息发送**：`yield event.plain_result("text")`
4. **配置读取**：`self.config.get("key", default)`
5. **数据存储**：存到 `data/` 目录，不要存插件目录
6. **异步网络**：使用 `aiohttp` 或 `httpx`，不要用 `requests`
7. **日志**：`from astrbot.api import logger`

### Step 3: 创建 requirements.txt

```
httpx>=0.24.0
```

### Step 4: 实现 main.py

1. **YuqueClient 类**：封装语雀 API
2. **/bind 指令**：
   - 参数：语雀 Token 或用户名
   - 验证 Token → 获取用户信息
   - 检查冲突 → 保存绑定
   - 触发画像生成
3. **/unbind 指令**：解除绑定
4. **/profile 指令**：查看画像

### Step 5: 数据结构

**bindings.json**（存到 `data/nova/`）：
```json
{
  "feishu_id": {
    "yuque_login": "username",
    "yuque_id": 123456,
    "yuque_name": "张三",
    "bind_time": "2026-03-26T01:30:00"
  }
}
```

## 冲突检测逻辑

1. **同一飞书账号已绑定**：提示解绑后重新绑定
2. **语雀账号被他人绑定**：警告并请求确认
3. **姓名模糊匹配多结果**：列出候选让用户选择

## 验证方式

1. 发送 `/bind` → 提示输入 Token 或姓名
2. 发送 `/bind <token>` → 验证并绑定
3. 发送 `/profile` → 查看绑定状态
4. 发送 `/unbind` → 解除绑定
