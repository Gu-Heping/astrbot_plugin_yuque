## 分析

### gno MCP 架构

根据设计决策（T000），NovaBot **不实现自己的检索引擎**，而是复用 gno MCP：

```
用户提问 → AstrBot LLM → 判断需要检索 → 调用 gno.query MCP 工具
          ↓
gno 返回相关文档 → LLM 生成回答 → 添加来源标注
```

### 关键发现

1. **MCP 是 AstrBot 级别配置**，不是 Plugin 级别
2. **Plugin 无需写检索代码**，gno MCP 自动注册工具
3. **LLM 自动发现并调用工具**

## 实现计划

### Step 1: 添加系统提示钩子

在 Plugin 中使用 `@filter.on_llm_request()` 钩子，添加系统提示引导 LLM 使用 gno 工具。

```python
@filter.on_llm_request()
async def on_llm_request(self, event, req):
    req.system_prompt += """

你是 NovaBot，NOVA 社团的智能助手。

当用户提问涉及技术文档、教程、项目经验时，请先使用 gno.query 工具搜索语雀知识库，然后基于搜索结果回答。

回答时请标注来源，例如：
- 「根据《文档名》by 作者...」
- 「相关文档：《...》」

如果用户问的是个人信息（如「我的画像」），引导使用 /profile 指令。
"""
```

### Step 2: gno MCP 配置说明

**AstrBot WebUI 配置**：

1. 打开 AstrBot WebUI
2. 进入 AI 配置 → MCP
3. 点击"新增服务器"
4. 填写配置：

```json
{
  "command": "/home/admin/.bun/bin/gno",
  "args": ["mcp", "serve"],
  "env": {
    "NODE_LLAMA_CPP_GPU": "false"
  }
}
```

5. 点击"测试"
6. 测试成功后点击"保存"
