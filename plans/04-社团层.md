### 1.2 数据来源

| 数据项 | 来源 | 说明 |
|--------|------|------|
| 热门文档 | **语雀 API** / 本地 frontmatter | 访问量需 API，更新时间可从本地获取 |
| 活跃作者 | 本地 frontmatter + Git diff | 对比上周的文档变更 |
| 趋势观察 | NovaBot 检索日志 | 记录用户的搜索关键词 |
| 优质内容 | LLM 评估 + 访问数据 | 结合内容质量与访问量 |
| 下周活动 | 飞书日历 / 手动配置 | 可选接入 |

**周报数据采集流程**：

```python
async def generate_weekly_report(start_date: str, end_date: str) -> dict:
    """生成周报"""
    
    # 1. 热门文档（从语雀 API 获取访问量）
    docs = await get_docs_with_stats()
    popular = sorted(docs, key=lambda x: x["views"], reverse=True)[:5]
    
    # 2. 活跃作者（对比本地文档变更）
    changes = detect_doc_changes(start_date, end_date)
    authors = aggregate_by_author(changes)
    
    # 3. 趋势分析（从检索日志分析）
    search_logs = load_search_logs(start_date, end_date)
    trends = analyze_trends(search_logs)
    
    # 4. 优质内容发现
    hidden_gems = find_hidden_gems(docs)
    
    return {
        "period": f"{start_date} - {end_date}",
        "popular_docs": popular,
        "active_authors": authors,
        "trends": trends,
        "hidden_gems": hidden_gems
    }
```

**简化方案（无语雀 API 访问量）**：

```python
# 如果无法获取访问量，用以下替代：
# - 文档更新频率
# - frontmatter 中的 likes（如果有）
# - NovaBot 检索命中次数
# - 评论数（如果有）

def rank_by_activity(docs: list) -> list:
    """按活跃度排序"""
    for doc in docs:
        score = 0
        score += doc.get("update_count", 0) * 2  # 更新次数
        score += doc.get("search_hits", 0)       # 检索命中
        score += len(doc.get("comments", []))    # 评论数
        doc["activity_score"] = score
    
    return sorted(docs, key=lambda x: x["activity_score"], reverse=True)
```

### 2.2 知识缺口发现

**识别方式**：

```python
def find_knowledge_gaps():
    """
    1. 分析无结果的搜索查询
    2. 找到高频但文档稀少的主题
    3. 识别文档覆盖不完整的知识卡片
    """
    
    gaps = []
    
    # 无结果查询
    for query in no_result_queries:
        if query.count > 5:  # 阈值
            gaps.append({
                "topic": query.text,
                "demand": query.count,
                "type": "no_docs"
            })
    
    # 稀疏主题
    for topic in knowledge_cards:
        if topic.doc_count < 2:
            gaps.append({
                "topic": topic.name,
                "demand": "low_coverage",
                "type": "sparse"
            })
    
    return gaps
```

## 3. 优质内容发现

### 3.1 识别标准

| 指标 | 权重 | 说明 |
|------|:----:|------|
| 内容质量 | 40% | LLM 评估的知识深度、实用性 |
| 访问量 | 25% | 相对同类型文档的访问量 |
