## 1. 存储方案选择

### 1.1 方案对比

| 方案 | 复杂度 | 适用场景 | NovaBot |
|------|:------:|----------|:-------:|
| MongoDB + Qdrant | 高 | 生产级、大规模 | ❌ |
| **Markdown + JSON** | **低** | **MVP、小团队** | ✅ |
| 纯内存 | 最低 | 单次运行 | ❌ |

### 1.2 选择理由

- yuque2git 已采用此方案，**实践验证可行**
- 413 篇文档，数据量小，JSON 足够
- 便于 Git 版本管理
- AstrBot Plugin 无需额外数据库依赖
- **可直接复用 yuque2git 的数据**

### 3.2 绑定冲突检测

**场景 1：一个飞书账号想绑定多个语雀账号**

```
用户 A（飞书 ID: ou_1）已绑定语雀账号 @zhangsan
用户 A 再次执行 /bind，想绑定 @lisi

处理：
❌ 拒绝绑定，提示：
"你的飞书账号已绑定 @zhangsan。
如需更换绑定，请先使用 /unbind 解绑。"
```

**场景 2：多个飞书账号绑定同一个语雀账号**

```
用户 A（飞书 ID: ou_1）已绑定语雀账号 @zhangsan
用户 B（飞书 ID: ou_2）执行 /bind，想绑定 @zhangsan

处理：
⚠️ 警告并确认：
"⚠️ 语雀账号 @zhangsan 已被另一个飞书账号绑定。
确认要绑定吗？（这会解除原绑定）

输入 /bind confirm 确认绑定"
```

**场景 3：模糊匹配姓名冲突**

```
用户输入：/bind 张三

系统检索 .yuque-members.json：
- 找到 2 个匹配：张三 (id: 123)、张三丰 (id: 456)

处理：
"找到多个匹配：
1. 张三 (@zhangsan)
2. 张三丰 (@zhangsanfeng)

请输入序号选择，或直接输入语雀用户名。"
```

**冲突检测代码**：

```python
async def check_binding_conflict(feishu_id: str, yuque_id: int) -> dict:
    """检查绑定冲突"""
    bindings = load_bindings()
    
    # 检查飞书账号是否已绑定
    if feishu_id in bindings:
        return {
            "conflict": True,
            "type": "feishu_already_bound",
            "current_binding": bindings[feishu_id]
        }
    
    # 检查语雀账号是否被其他飞书账号绑定
    for fid, info in bindings.items():
        if info["yuque_id"] == yuque_id and fid != feishu_id:
            return {
                "conflict": True,
                "type": "yuque_already_bound",
                "bound_by": fid,
                "bound_by_name": info.get("yuque_name")
            }
    
    return {"conflict": False}
```

## 4. 复用 yuque2git 数据

### 4.1 可直接使用的数据

| yuque2git 文件 | NovaBot 用途 |
|----------------|-------------|
| `.yuque-id-to-path.json` | 文档 ID → 文件路径，快速定位文档 |
| `.repos.json` | 知识库列表、元数据 |
| `.yuque-members.json` | 团队成员信息 |
| 文档 frontmatter | 作者、时间、知识库、描述 |
| 文档 `.toc.json` | 知识库目录结构 |

### 4.2 文档 frontmatter 格式

```yaml
---
id: 262764261              # 文档 ID
title: 项目五选 x           # 标题
slug: choda2kozxenn4ue     # URL slug
created_at: '2026-03-25 09:33:05'
updated_at: '2026-03-25 10:32:42'
author: 黄亮               # 作者
book_name: 兴趣组列表       # 知识库名
description: 序言上学期...  # 描述
---
```

### 4.3 查询示例

**查找某用户写的所有文档**：

```python
# 1. 从 bindings.json 获取用户的 yuque_name
user_name = bindings[feishu_id]["yuque_name"]

# 2. 遍历所有 md 文件，检查 frontmatter.author
docs = []
for md_file in yuque_docs.glob("**/*.md"):
    fm = parse_frontmatter(md_file)
    if fm.get("author") == user_name:
        docs.append({
            "id": fm["id"],
            "title": fm["title"],
            "path": md_file
        })
```

## 6. 数据一致性

### 6.1 增量更新策略

```
1. 检查 .yuque-id-to-path.json 中是否有新文档 ID
2. 对比 frontmatter.updated_at 与上次同步时间
3. 只处理变更的文档
```

### 6.2 知识卡片增量更新

```
1. 文档变更时，找出关联的知识卡片
2. 只重新生成受影响的部分
3. 保留用户的 personal_notes
```
