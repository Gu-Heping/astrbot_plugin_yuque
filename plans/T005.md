# T005: 集成 LangChain + ChromaDB RAG

## 目标
实现内置 RAG 检索模块，替代 gno MCP 依赖

## 技术选型

### Embedding 服务
- 默认使用 DashScope（通义千问 embedding）
- 支持配置 OpenAI 兼容 API

### 向量存储
- ChromaDB（轻量级，无需额外服务）
- 数据存储在 `data/nova/chroma/`

### 文档处理
- LangChain TextSplitter
- Markdown 优化分割

## 实现步骤

1. **RAGService 类**
   - 初始化 ChromaDB 客户端
   - 配置 embedding 函数
   - 提供索引和检索接口

2. **文档索引**
   - 从 yuque_docs 目录读取同步的文档
   - 分割、生成 embedding、存储

3. **检索接口**
   - `/search <query>` 指令
   - LLM 工具：`search_knowledge`

4. **配置项**
   - embedding_api_key
   - embedding_model
   - chunk_size / chunk_overlap

## 文件变更
- `main.py` - 添加 RAGService 和相关指令
- `requirements.txt` - 添加依赖
- `_conf_schema.json` - 添加配置项

## 验收标准
- [ ] 能索引同步的文档
- [ ] 能检索并返回相关文档
- [ ] LLM 能调用 search_knowledge 工具
